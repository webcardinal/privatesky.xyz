Pskwallet is a command line tool for creating and manipulating dossiers. The create dossier command creates a dossier with the provided name and prints its SEED. pskwallet create DSU dossierName constitutionPath [nosave] dossierName The name of the DSU that is being created.

constitutionPath The file system path where the bundles that form the DSU's constitution are located. When the create DSU command is executed, the files located at constitutionPath are added to the created DSU.

nosave | Optional A flag that establishes whether the created DSU is mounted inside the current wallet. If nosave is present, then the created DSU is not mounted inside the current wallet; if present, then the created DSU is mounted inside the wallet at path /code/constitution/dossierName. A DSU created using the create DSU command without the nosave flag is called a template DSU. The create wallet command creates a DSU that contains, mounted at path /code/constitution another DSU created using the create dossier command without the nosave flag. When the create wallet command is executed, a prompt appears requiring the user to insert a password. The password is used as a cryptographic key to encrypt the wallet's SEED. The encrypted SEED is stored in the local file system, at path /home/.seedCage. If the file exists, from a previous wallet, a prompt appears asking the user if the file can be overwritten. Note: After creating a wallet, any subsequent command that involves the wallet will require the password for authentication. pskwallet create wallet templateSeed templateSeed The SEED of a template dossier. The restore command restores the wallet associated with the provided SEED. When the restore command is executed, a prompt appears requiring the user to insert a password. The password is used as a cryptographic key to encrypt the wallet's SEED. The encrypted SEED is stored in the local file system, at path /home/.seedCage. If the file exists, from a previous wallet, a prompt appears asking the user if the file can be overwritten. Note: After restoring a wallet, any subsequent command that involves the wallet will require the password for authentication. pskwallet restore seed seed The SEED of the restored wallet. The set app command adds an app to a previously created dossier. An app is folder in the local file system that contains all the code and data of an application. The app is added in the target DSU at path /code/app. pskwallet set app [seed] [alias] folderPath seed | Optional The SEED of the DSU in which the app is added.

alias | Optional The name that is given to a DSU when adding it to a wallet. This name is used to reference the dossier inside the wallet. Thus, an app can be added to a DSU mounted inside the wallet.

Note: At least one of the parameters seed and alias should be specified. folderPath The file system path where the folder that contains the app is located. The get app command creates a extracts an app from a DSU to a folder in the local file system. pskwallet get app [seed] [alias] barPath fsFolderPath seed | Optional The SEED of the DSU in which the app is added.

alias | Optional The name of the dossier, mounted inside the wallet, from which the app is extracted.

Note: At least one of the parameters seed and alias should be specified. barPath The path inside the DSU where the app is located.

fsFolderPath The file system path where the app is extracted to. The mount command mounts a DSU (source) inside another DSU (destination). pskwallet mount [destinationSeed] [alias] path sourceSeed destinationSeed | Optional The destination dossier's SEED.

alias | Optional The destination dossier's name, if it is mounted inside the wallet.

Note: At least one of the parameters seed and alias should be specified. path The path inside the destination DSU where the source DSU is mounted.

sourceSeed The source dossier's SEED. More information about the mount operation can be found here. The unmount command deletes the reference of a DSU (source) from another DSU (destination) inside which was previously mounted. pskwallet unmount [destinationSeed] [alias] path destinationSeed | Optional The destination dossier's SEED.

alias | Optional The destination dossier's name, if it is mounted inside the wallet.

Note: At least one of the parameters seed and alias should be specified. path The path inside the destination DSU where the source DSU is mounted. More information about the mount operation can be found here. The list mounts command prints a list of all SEEDs and names of the dossiers mounted inside a destination dossier. pskwallet list mounts [destinationSeed] [alias] path destinationSeed | Optional The destination dossier's SEED.

alias | Optional The destination dossier's name, if it is mounted inside the wallet.

Note: At least one of the parameters seed and alias should be specified. path The path inside the destination DSU from which the search begins. More information about the mount operation can be found here.
