WebCardinal configuration in made via webcardinal.json.

The initial settings are read from the default.ts.

Each page from pages/ must be registered.

pages: [ ... { /* required */ "name": "My page" /* optional */ "path": "/my-custom-path" // default value would have been "./my-page" "src": "./my-custom-page.html" // default: "./my-page.html" "indexed": false // default: true "children": [...] // array of multiple pages "tag": "my-page" // used by webc-link or navigateByPageTag from any Controller } ... ]

src and path are relative to basePath and parent page.

If window.location.href does not match anything, application will use this page.

Concept of 404 Not Found.

pagesFallback: { /* required */ "name": "My Fallback Page" /* optional */ "src": "./404.html" }

Configure application name, email and logo.

Read more about these settings in the documentation for webc-app-identity.
