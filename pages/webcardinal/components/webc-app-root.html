The entry point of your application.

This component has attached a controller called ApplicationController that converts the configurations from webcardinal.json into a WebCardinal application. Default Configuration of WebCardinal can and should be modified.

webc-app-root contains by default:

  • webc-app-menu
  • webc-app-container
  • webc-app-error-toast

Notes:

Components that have in name webc-app are internal and are not part of a normal page. ApplicationController responds to all events that are prefixed with webcardinal:config.

webc-app-menu and webc-app-container can be manged using slots, but webc-app-error-toast will be always appended.

The element is accessible from window.WebCardinal.root.

You must place this component in index.html.

Responsiveness of components is manged by the webc-app-root itself.

In this example example layout of webc-app-root is managed by webc-app-menu.

Where custom-header, custom-main, custom-footer are personalized / customized web components.

You must deal with responsiveness of these components.
