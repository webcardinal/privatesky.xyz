This is the entry point of your application and contains by default:

  • webc-app-menu
  • webc-app-container
  • webc-app-error-toast

Note: webc-app-menu and webc-app-container must be manged using slots, but webc-app-error-toast will be always appended.

It is also worth mentioning that this component invokes ApplicationController.

Default Configuration of WebCardinal can and should be modified from webcardinal.json.

You must place this component in index.html.

Responsiveness of components is manged by the webc-app-root itself.

In this example example layout of webc-app-root is managed by webc-app-menu.

Where custom-header, custom-main, custom-footer are personalized / customized web components.

You must deal with responsiveness of these components.
