In order to build a new design system for a project that uses Cardinal, we need to create a new theme. By adding a new theme to the project, all web components provided by CardinalJs can be customized according to the desired design.

By default, Cardinal has a folder named “themes” where we can find a standard theme for the web components. The first step in creating a new theme is to add a new theme folder. For example, besides the default theme, we can add a new folder called “newlook”.

Every theme folder contains an assets and a components subfolder. In the assets we can add different fonts, icons, images or libraries that are necessary for the new design system. The assets subfolder can be empty, but the components subfolder is absolutely necessary in order to make the new theme. In “components” we will have to add a folder for every web component and assign to each one a css file with the same name. This will help us customize each web component separately by changing the css properties according to the new design system.

While designing web components an important thing to take into consideration is the Shadow DOM. If the Shadow DOM of a web component is enabled, elements within the shadow root are scoped and all the styles outside of the component do not apply. This means that we have to style the elements inside the Shadow Dom from within the component.

Normally, you could add your styles in the tag name of the component just like in the example below:

my-element { color: black; } my-element p { color: blue; }

With Shadow DOM, CSS selectors inside the component can be simplified and the parent component can be styled using the :host selector. Therefore the code above could be:

:host { color: black; } p { color: blue; }

The :host selector can be in handy, if you want to add style to elements based on the parent component attributes. For example, the app-menu component uses in its composition the psk-menu-item-renderer component. Using the app-menu attribute named hamburger, we can write in psk-menu-item-renderer.css file new css rules that can change the children elements of the app-menu component. In the example below, you can see how to change all the div elements width, if the attribute hamburger is set up to true:

:host([hamburger]) div { width: 300px; }

As we said above in order to customize a web component we need to change the css file that has the same name as the component. Through css we can change the layout structure, color schemes, topography styles and others. Below you can find examples on how to customize different web components.

An important thing to remember about the psk-card is that this component is used to build up other components such as psk-chapter, psk-toc, psk-description or others. If we want to have the same design for all the psk-card components we will have to only change the psk-card.css file. If we want a different psk-card style based on the parent component we will have to change the parent css file ( for example psk-chapter.css).

For customizing the psk-code component we can use PrismJS . Prism is a lightweight, extensible syntax highlighter, built with modern web standards, that can help us generate all the css we need. We have the option of choosing between downloading the developer version or the minified one. Prism also offers us multiple themes for the code style of a lot of programming languages. By downloading the developer version you can easily modified the predefined theme to make it match your design.
